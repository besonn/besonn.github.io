Mips学习2

· 2020-04-24 · # 计算机组成原理 # misp

今天的内容主要是函数和过程调用

函数

.text
	main:
		jal displayMessage
		
		addi $s0,$zero,5
		
		# Print number 5 to the screen
		li $v0,1
		add $a0,$zero,$s0
		syscall
		
	# Tell the system the program is done
	li $v0,10 # 看一下Mips手册找一下条件
	syscall 
	
	displayMessage:
		li $v0,4
		la $a0,message
		syscall 
		jr $ra

调用函数：jal 函数名；
函数内部中：jr ra跳转回原来地址ra 跳转回原来地址 ra跳转回原来地址ra 中用来保存原函数的返回地址

函数的传参

.data

.text
	main:
		addi $a1,$zero,100
		addi $a2,$zero,59
		
		jal addNumbers
		
		li $v0,1
		add $a0,$zero,$v1
		syscall
		
	# Tell the system that the program is done.
	li $v0,10
	syscall
	
	addNumbers:
		add $v1,$a1,$a2
		jr $ra

函数中的传参：a寄存器可以作为子函数的参数寄存器，一般在main函数里面都是$a0寄存器返回结果。子函数结果返回用v寄存器。

s寄存器的使用

.data
	newLine: .asciiz "\n"
.text
	main:
		addi $s0, $zero, 10
		jal addNumbers
	
		# Print Value
		li,$v0,4
		la $a0,newLine
		syscall
		li,$v0,1
		move $a0,$s0
		syscall
	
	# Tell the system the program is done.
	li $v0,10
	syscall
	
	addNumbers:
		addi $sp,$sp,-4
		#sw $t0, 0($sp)
		
		addi $s0,$s0,40
		
		#Print new value in function
		
		li $v0,1
		move $a0,$s0
		syscall
		
		#lw $t0,0($sp)
		addi $sp,$sp,4
		
		jr $ra

通过s寄存器观察过程调用前后值的变化，s寄存器在传入子函数前后需要保证值是一样的，所以在子函数中，需要用栈帧sp来保存一下s寄存器中的值，在结束之后，将栈帧中保存的值返回到s当中去.

进入子函数需要拓展一下存放参数地址的空间。

与其相对的，t寄存器的值可以任意使用，只不过这样就不能保证里面存放的值是否是之前的了。

函数的嵌套

.data
	newLine: .asciiz "\n"
.text
	main:
		addi $s0, $zero, 10
		jal addNumbers
		li,$v0,4
		la $a0,newLine
		syscall
		# Print Value
		jal printValue
	
	# Tell the system the program is done.
	li $v0,10
	syscall
	
	addNumbers:
		addi $sp,$sp,-8
		sw $s0, 0($sp)
		sw $ra, 4($sp)
		
		addi $s0,$s0,40
		
		#Print new value in function
		jal printValue
		
		lw $s0,0($sp)
		lw $ra,4($sp)
		addi $sp,$sp,8
		
		jr $ra
	printValue:
		li,$v0,1
		move $a0,$s0
		syscall
		
		jr $ra

函数的嵌套需要注意一点：被嵌套的子函数需要将其返回地址一样的保存到其上一级函数的栈帧当中，因此调用函数需要增加4个空间的值。addi $sp,$sp,-8 然后再保存返回地址到栈帧当中去，sw $ra, 4($sp)

最后不要忘记了释放掉栈帧中的空间。
